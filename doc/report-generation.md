# EDGESCRIBE Report Generation — v1.2 Feature Design

## Overview

Generate comprehensive multi-page clinical reports by assembling data from multiple sources (transcripts, SOAP notes, lab results, images) using **sectional generation** — each section generated independently by the LLM for maximum quality.

**Key constraint**: A 2B model degrades after ~2,000 tokens of continuous generation. The solution is to generate each section as a focused, short task (200-800 tokens each), then stitch them together into a complete report.

## Sectional Generation (Map-Reduce)

### Why not one-shot?

```
One-shot: "Write a 10-page report from this transcript"
  → First 2 pages: good
  → Pages 3-5: repetitive, losing structure
  → Pages 6-10: hallucinating, contradicting earlier sections
  → Total: ~7,000 tokens, poor quality

Sectional: Generate each section independently
  → Section 1 (200 tokens): excellent — focused extraction
  → Section 2 (500 tokens): excellent — short narrative
  → Section 3 (400 tokens): excellent — structured list
  → ... 8 more sections, each excellent
  → Assemble: ~4,000 tokens, high quality throughout
```

### Generation Pipeline

```
Step 1: Gather inputs
  ├── transcript.txt (from ASR)
  ├── soap_notes.txt (from process --soap)
  ├── lab_results.txt (user-provided)
  ├── image analysis (from vision engine)
  └── previous sessions (from memory store)

Step 2: Generate each section (8-11 LLM calls, ~5s each)
  ├── Demographics:  extract from transcript
  ├── Chief Complaint: extract from SOAP subjective
  ├── HPI: rewrite transcript into medical narrative
  ├── Past Medical Hx: extract mentions from transcript
  ├── ROS: extract system-by-system from transcript
  ├── Physical Exam: structure from SOAP objective
  ├── Lab Results: summarize provided data
  ├── Assessment: from SOAP assessment + reasoning
  ├── Treatment Plan: from SOAP plan
  ├── Patient Education: generate based on assessment
  └── Follow-up: generate based on plan

Step 3: Assemble sections into final report

Step 4: Save to history + export
```

### Per-Section Prompts

Each section gets a focused prompt with ONLY the relevant context:

```
Section: HPI
System: "You are a medical report writer. Write the History of Present
         Illness section based on the transcript below. Use formal medical
         language, third person, past tense. 1-2 paragraphs."
Input:  [transcript text]
Output: ~300-500 tokens

Section: Assessment
System: "You are a medical report writer. Write the Assessment section
         including differential diagnoses based on the SOAP assessment
         and examination findings below. List diagnoses with ICD-10 codes
         if identifiable."
Input:  [SOAP assessment + objective sections]
Output: ~400-800 tokens
```

## Context Budget Per Section

Total context: 16,384 tokens. Each section uses only what it needs:

| Section | Input tokens | Output tokens | Total |
|---|---|---|---|
| Demographics | ~200 (transcript excerpt) | ~200 | ~400 |
| Chief Complaint | ~500 (SOAP subjective) | ~200 | ~700 |
| HPI | ~2,000 (full transcript) | ~500 | ~2,500 |
| Past Medical Hx | ~2,000 (full transcript) | ~300 | ~2,300 |
| ROS | ~2,000 (full transcript) | ~400 | ~2,400 |
| Physical Exam | ~800 (SOAP objective) | ~500 | ~1,300 |
| Lab Results | ~1,000 (lab data + image OCR) | ~600 | ~1,600 |
| Assessment | ~1,500 (SOAP + exam + labs) | ~800 | ~2,300 |
| Treatment Plan | ~1,000 (SOAP plan) | ~600 | ~1,600 |
| Patient Education | ~800 (assessment summary) | ~400 | ~1,200 |
| Follow-up | ~500 (plan summary) | ~300 | ~800 |

**Every section fits comfortably in 16K context.** No section exceeds ~2,500 tokens total.

### Total Generation

```
Sum of all output tokens: ~4,800 tokens ≈ 3,500 words ≈ ~7-10 pages
Total LLM calls: 11
Total time (CPU): ~55 seconds (11 × ~5s each)
Total time (GPU/Vulkan): ~15 seconds (11 × ~1.5s each)
```

## CLI Interface

```bash
# Generate report from transcript + SOAP
edgescribe report \
  --transcript transcript.txt \
  --soap soap_notes.txt \
  -o report.txt

# With lab results and imaging
edgescribe report \
  --transcript transcript.txt \
  --soap soap_notes.txt \
  --labs lab_results.txt \
  --image xray.jpg \
  -o report.txt

# From most recent session (auto-pulls from memory)
edgescribe report --latest -o report.txt

# Using the clinical workflow (record → SOAP → report)
edgescribe workflow clinical --live --report -o report.txt

# Choose template
edgescribe report --template discharge-summary --transcript ...
edgescribe report --template consultation --transcript ...
edgescribe report --template progress-note --transcript ...

# Export formats
edgescribe report ... -o report.txt         # Plain text
edgescribe report ... -o report.md          # Markdown
edgescribe report ... --format pdf -o report.pdf  # PDF (future)
```

### Template Definition

```cpp
struct ReportTemplate {
    std::string name;
    std::string display_name;
    std::vector<SectionDef> sections;
};

struct SectionDef {
    std::string id;             // "hpi", "assessment", etc.
    std::string title;          // "History of Present Illness"
    std::string system_prompt;  // Instructions for the LLM
    std::string input_source;   // "transcript", "soap", "labs", "image"
    int max_tokens = 500;       // Output limit for this section
};
```

## Implementation

### ReportGenerator Class

```cpp
namespace EDGESCRIBE {

class ReportGenerator {
 public:
    ReportGenerator(llm::LlmEngine* llm,
                    vision::VisionEngine* vision = nullptr);

    struct ReportInput {
        std::string transcript;
        std::string soap_notes;
        std::string lab_results;
        std::string image_path;
        std::string template_name = "standard";
    };

    struct ReportOutput {
        std::string full_report;
        std::map<std::string, std::string> sections;
        double generation_time_ms;
    };

    ReportOutput Generate(const ReportInput& input,
                          ProgressCallback on_section = nullptr);

    std::vector<std::string> ListTemplates() const;

 private:
    std::string GenerateSection(const SectionDef& section,
                                const ReportInput& input);

    llm::LlmEngine* llm_;
    vision::VisionEngine* vision_;
    std::map<std::string, ReportTemplate> templates_;
};

}  // namespace EDGESCRIBE
```

### Section Generation

```cpp
std::string ReportGenerator::GenerateSection(const SectionDef& section,
                                              const ReportInput& input) {
    // Select input based on section definition
    std::string context;
    if (section.input_source == "transcript") {
        context = input.transcript;
    } else if (section.input_source == "soap") {
        context = input.soap_notes;
    } else if (section.input_source == "labs") {
        context = input.lab_results;
    } else if (section.input_source == "all") {
        context = "Transcript:\n" + input.transcript +
                  "\n\nSOAP Notes:\n" + input.soap_notes;
        if (!input.lab_results.empty())
            context += "\n\nLab Results:\n" + input.lab_results;
    }

    // Truncate context if too long (keep last N tokens)
    // Each section gets focused context, not everything

    return llm_->Chat(section.system_prompt, context, section.max_tokens);
}
```

### Report Assembly

```cpp
ReportOutput ReportGenerator::Generate(const ReportInput& input,
                                        ProgressCallback on_section) {
    ReportOutput output;
    auto t_start = std::chrono::steady_clock::now();

    const auto& tmpl = templates_[input.template_name];
    std::ostringstream report;

    report << "COMPREHENSIVE CLINICAL REPORT\n";
    report << std::string(50, '=') << "\n\n";

    for (size_t i = 0; i < tmpl.sections.size(); i++) {
        const auto& section = tmpl.sections[i];

        if (on_section) on_section(section.title, "generating");

        std::string text = GenerateSection(section, input);
        output.sections[section.id] = text;

        report << (i + 1) << ". " << section.title << "\n";
        report << std::string(40, '-') << "\n";
        report << text << "\n\n";

        if (on_section) on_section(section.title, "done");
    }

    report << std::string(50, '=') << "\n";
    report << "DRAFT — Requires physician review and signature\n";
    report << "Generated by EDGESCRIBE on "
           << CurrentTimestamp() << "\n";

    output.full_report = report.str();
    output.generation_time_ms = ElapsedMs(t_start);

    return output;
}
```

```

DRAFT — This report was generated by AI and requires
physician review, verification, and signature before
becoming part of the medical record.
```

### Section-level confidence (future)

Flag sections where the model might be less reliable:

```
8. Assessment & Differential Diagnosis
   ⚠️ AI-generated differential — verify against clinical judgment
   
   Primary diagnosis: Unstable angina (ICD-10: I20.0)
   Differential:
   - Acute coronary syndrome
   - Costochondritis
   - GERD with atypical presentation
```
